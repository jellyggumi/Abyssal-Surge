# Shadow Lord RTS-RPG Research Plan

## Main question
What production, game-design, and deployment choices let Abyssal Surge evolve its existing browser campaign into an original, mobile-first Shadow Lord RTS-RPG hybrid while retaining a credible staged path to persistence, multiplayer, audio-visual assets, GitHub Pages, and later Android packaging?

## Evidence contract
- Research starts from the existing `_workspace/` production artifacts and LLM wiki.
- External claims use primary sources where available; every source URL is retained.
- This research does not authorize replacement of the deployed browser campaign or unverified third-party asset use.
- Recommendations must distinguish the initial offline static release from a later server-backed multiplayer evolution.

## Subtopics
1. **Comparable-game design benchmarks** — analyze RTS-RPG hybrids and adjacent mobile strategy games for touch controls, tactical-loop cadence, campaign progression, and balance-validation patterns.
2. **Mobile web, PWA, and Android delivery** — determine static-host-friendly architecture, offline/PWA constraints, testable GitHub Pages delivery, and the viable TWA/APK route after web validation.
3. **Persistence and multiplayer evolution** — compare authoritative backend approaches for an initially static game, including minimal data model, anti-cheat boundaries, real-time transport, and migration triggers.
4. **Original multimedia production pipeline** — assess documented, rights-safe interfaces and deliverable formats for image, mesh/animation, audio/narration, cinematic video, and browser fallbacks.
5. **Iterative governance** — map numeric gates, Pydantic-validated reflection outputs, evidence artifacts, and wiki updates into an executable three-stage game-production loop.

## Synthesis
Synthesize bounded findings into a source-linked production contract under the LLM wiki and `_workspace/`, then feed the frozen requirements to the Game Studio Harness and the OOO seed. Recommendations must name acceptance evidence and identify decisions that need user input rather than inventing commitments.
