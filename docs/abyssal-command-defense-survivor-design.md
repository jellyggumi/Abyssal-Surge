# Abyssal Command 디펜스 서바이버 설계

## 제품 정의

Abyssal Command는 모바일 우선의 싱글플레이 디펜스 서바이버다. 플레이어는 생존자를 이동시키고, 기본 전투는 자동으로 해결한다. 목표는 한 런의 XP 빌드와 영구 동료 수집을 조합해 10개 스테이지 캠페인을 끝내는 것이다. 이 문서는 현재 저장소에 배포되는 유일한 제품 계약이다.

## 화면과 조작

- 전장은 full-bleed 모바일 Canvas로 표시한다. 상태 정보와 조작은 가장자리 HUD에 두고, 적·투사체·위험 영역을 덮는 중앙 패널은 사용하지 않는다.
- 런타임은 가능한 경우 fullscreen과 landscape lock을 자동 요청한다. 권한·브라우저·기기 제한으로 잠금할 수 없는 경우 회전 프롬프트를 표시하지 않는다. 세로 뷰포트에서는 시계 방향으로 회전한 논리 가로 화면을 그대로 투영한다.
- 플레이어 입력은 이동에 집중한다. 기본 공격과 대상 선택은 자동이며, 조준·공격·전술 명령 대기열을 요구하지 않는다.
- 접근성 설정과 `prefers-reduced-motion`을 존중한다. 저감 모드에서는 연속 흔들림·번쩍임·장식 애니메이션을 줄이고, 위험·피해·선택 상태는 정적인 대비와 텍스트로도 읽을 수 있어야 한다.

## 런과 성장

1. 적을 피하고 자동 기본 전투로 XP를 얻는다.
2. XP 임계값마다 런 전용 스킬 제안을 제시하고 하나를 선택한다. 선택하지 않은 제안은 이번 선택에서 사라지며, 런 종료 후 일반 스킬 효과는 남지 않는다.
3. 정예 적 처치 뒤에는 추출 기회가 있다. 성공한 추출은 영구 동료를 해금하며 이후 런과 캠페인에 사용할 수 있다.
4. 스테이지 보스 승리는 다음 스테이지를 연다. 패배는 현재 스테이지 런을 다시 시작하며 런 전용 스킬은 초기화한다.
5. Stage 10 보스 승리는 캠페인 완료다.

영구 동료와 런 전용 스킬은 별도 상태다. 동료 획득은 로컬 영구 진행에 기록하지만, XP 레벨·제안·스킬 효과·현재 전투 배치는 런 범위를 벗어나 보존하지 않는다.

## 시뮬레이션과 저장

- 게임 규칙은 고정 시간 간격의 결정론적 60 Hz 시뮬레이션으로 실행한다. 렌더 프레임률은 규칙 tick 수를 바꾸지 않는다.
- 재현에 필요한 상태와 입력 순서를 저장한다. 시간·난수·동률 처리는 명시적으로 안정화하여 같은 저장 상태와 입력이 같은 결과를 낸다.
- 저장은 오프라인 로컬 우선이다. 브라우저 로컬 영구 저장과 JSON 내보내기/가져오기를 제공한다. 네트워크, 계정, 클라우드 동기화, 멀티플레이를 요구하지 않는다.

## 투영 계약

현재 배포 경로는 Canvas 2D 스냅샷 투영이다. 전장 어댑터는 결정론적 시뮬레이션 상태만 읽고, 렌더러 오류 시 같은 스냅샷 계약의 대체 어댑터가 표시를 이어간다. 투영 방식의 차이가 전투 결과·성장 제안·추출·캠페인 진행을 바꾸면 안 된다.
